# Динамическое программирование

**Динамическим программированием** называют подход к решению задач, при котором происходит рекурсивное разбиение исходной сложной задачи на перекрывающиеся более простые подзадачи. Задачи, в которой такое разбиение возможно, называются **задачами с оптимальной подструктурой**. Оптимальная подструктура подразумевает, что оптимальное решение подзадач меньшего размера может быть использовано для решения исходной сложной задачи. Перекрывающиеся подзадачи подразумевают, что их решения используются для решения нескольких (не одной) подзадач большего размера. Используя **мемоизацию** (запоминание решения данной подзадачи) или заранее просчитывая все подзадачи и собирая из них решение сложной задачи, мы получаем выигрыш от использования динамического программирования по сравнению с полным перебором.

## Кратчайшие пути в DAG

Рассмотрим задачу нахождения кратчайших путей в *ориентированном* ациклическом графе (directed acyclic graph, DAG) с положительными и отрицательными весами из некоторой вершины A.

![dag](images/oriented_graph_from_maze_dp_sp_1.svg)

Возьмем произвольную вершину, например, J. Расстояние до нее полностью определяется расстояниями до смежных вершин входящих ребер (E, I и K в данном случае) и весами этих ребер:

```
dist[J] = min(dist[E] + 1, dist[I] + 10, dist[K] + 1)
```

В данном случае оптимальное решение подзадачи для J (сложная подзадача) включает в себя решение подзадач для E, I и K (более простые подзадачи):

```
dist[E] = dist[B] + 2
dist[I] = dist[E] + 3
dist[K] = dist[B] + 3
```

Обратите внимание, что подзадачи I и J являются перекрывающими: они обе требуют решения для E, но при этом J требует дополнительно решения для K. Выигрыш от динамического программирования будет заключаться в том, что, найдя решение для E, мы запомним его и потом при решении следующей перекрывающей задачи возьмем уже готовое решение.

## Кратчайшие пути между всеми парами вершин в DAG (алгоритм Флойда-Уоршелла)

Для нахождения кратчайших путей между всеми парами вершин мы могли бы запустить алгоритм выше $|V|$ раз, что дало бы сложность $O(|E| |V|^2)$. Так как $|E| = O(|V|^2)$, это дает $O(|V|^4)$. Алгоритм Флойда-Уоршелла позволяет решить ту же задачу за $O(|V|^3)$ и, строго говоря, применим во всем взвешенным ориентированным графам без отрицательных циклов. Отметим, что применение алгоритма Дейкстры $|V|$ раз дает сложность $O(|V| |E| \log |V|)$, что лучше алгоритма Флойда-Уоршелла в случае разреженных графов ($|E| \ll |V|^2$)

Чтобы построить подзадачи в алгоритме Флойда-Уоршелла, рассмотрим построение кратчайших путей от соседей до наиболее удаленных друг от друга вершин.
Предположим, что все вершины в $V$ пронумерованы от 1 до $|V|$.
Заведем трехмерный массив `dist[i,j,k]`, который будет хранить кратчайшие расстояния от вершины `i` до вершины `j` при рассмотрении подмножества вершин `{1, 2, ..., k}` в качестве промежуточных.
Простейшие подзадачи формулируются для `k = 0`, так как в этом случае кратчайшими путями могут быть только ребра смежных вершин (если между `i` и `j` нет ребра, то `dist[i,j,k] = inf`).
Далее последовательно рассматриваются все `k` и обновляется `dist[i,j,k]` в том случае, если добавление `k` в качестве новой промежуточной вершины к уже рассмотренному подмножеству промежуточных вершин `{1, 2, ..., k - 1}` дает меньшее расстояние:

```
dist[i, k, k - 1] + dist[k, j, k - 1] < dist[i, j, k - 1]
```

Рассмотрим алгоритм Флойда-Уоршелла на примере. Ниже показана конфигурация графа и структуры `dist` (в данном случае это словарь) для начального состояния.

![fw1](images/oriented_graph_small_floyd_warshall_1.svg)

Первой промежуточной вершиной будет A. Ее добавление не открывает никаких дополнительных путей, следовательно, состояние `dist` не меняется.

![fw2](images/oriented_graph_small_floyd_warshall_2.svg)

Второй промежуточной вершиной будет B. Ее добавление открывает новый путь A -> B -> C, но так как его расстояние `dist[A, B, {A}] + dist[B, C, {A}] = 2` больше текущего наилучшего расстояния `dist[A, C, {A}] = 1`, мы не обновляем `dist`.

![fw3](images/oriented_graph_small_floyd_warshall_3.svg)

Третьей промежуточной вершиной будет C. Ее добавление открывает три новых пути: A -> C -> D (он улучшает расстояние между A и D с 5 до 3), A -> C -> E (первый путь между A и E)  и B -> C -> E (первый путь между B и E).

![fw4](images/oriented_graph_small_floyd_warshall_4.svg)
